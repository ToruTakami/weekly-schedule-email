# 週間予定のメール送信 - 要件ドキュメント

## プロジェクト概要

Google Calendarの予定を毎週自動取得し、メール本文に記載して定期送信するシステムを構築する。

---

## 1. 機能要件

### 1.1 スケジュール実行
- **実行頻度**: 毎週月曜日の08:00（UTC+9 日本時間）
- **実装方法**: GitHub Actions + Cowork連携

### 1.2 カレンダーデータ取得
- **取得元**: Google Calendar（Cowork側で既に認証済み）
- **取得範囲**: 実行日（月曜日）から7日間分の予定
- **取得項目**:
  - イベント名
  - 開始日時
  - 終了日時
  - 場所
  - 説明（備考）

### 1.3 メール本文作成
- **形式**: テキスト形式
- **期間表示**: 「2026/3/21〜2026/3/27」形式
- **具体例**:
  ```
  高見徹の１週間の予定（2026年3月21日〜3月27日）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 3月21日（土）
  予定なし

■ 3月22日（日）
  予定なし

■ 3月23日（月）
  17:00〜18:00　リハビリ（羊ケ丘病院付属リハビリクリニック）
　担当療法士：福土 百合子
　患者番号：00100151
　電話番号：011-351-0881
　備考：予約時間の15分前にご来院ください

■ 3月24日（火）
  予定なし

■ 3月25日（水）
  予定なし

■ 3月26日（木）
  予定なし

■ 3月27日（金）
  予定なし

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
以上です。
  ```

### 1.4 メール送信
- **件名**: `高見徹の１週間の予定（2026/3/21〜2026/3/27）`
- **本文**: 上記1.3で作成したテキスト
- **送信先**: takami.sp@gmail.com
- **送信方法**: Gmail API（Cowork側で既に認証済み）

## 2. 非機能要件

### 2.1 検証・安全性
- **送信前チェック**: 送信先アドレスが `takami.sp@gmail.com` であることを確認
- **エラーハンドリング**: 各ステップで失敗時の処理を定義
- **実行ログ**: GitHub Actions上に詳細な実行ログを記録

### 2.2 信頼性
- カレンダーデータ取得失敗時の対応
- メール送信失敗時の対応
- 送信前の検証失敗時の対応

### 2.3 監視・通知
- 実行結果（成功/失敗）をGitHub Actions上で確認可能にする
- 失敗時の詳細エラーログを記録

## 3. システム構成

### 3.1 実装構成図

```
GitHub Actions (毎週月曜08:00)
    ↓
[1] Cowork API経由でカレンダー取得リクエスト
    ↓
Cowork（Claude側）
    ├─ Google Calendar APIでデータ取得
    ├─ メール本文を生成
    └─ JSONで結果を返却
    ↓
[2] GitHub Actions でメール本文受け取り
    ↓
[3] 送信先の検証
    - takami.sp@gmail.com の確認
    ↓
[4] Gmail API でメール送信（Cowork側で実施または GitHub Actions側で実施）
    ↓
[5] 実行ログを記録
```

### 3.2 使用技術
- **スケジューラー**: GitHub Actions（Workflow Scheduler）
- **カレンダーAPI**: Google Calendar API（Cowork側）
- **メール送信**: Gmail API（Cowork側）
- **データ連携**: HTTP通信（JSON形式）
- **言語**: Python + Bash

## 4. 実装詳細

### 4.1 GitHub Actions Workflow
- **ワークフロー名**: `weekly-schedule-email`
- **トリガー**: `schedule: '0 23 * * 0'`（毎週日曜UTC 23:00 = 月曜JST 08:00）
- **処理フロー**:
  1. Cowork APIを呼び出し、カレンダーデータ取得
  2. 返却されたメール本文を取得
  3. 送信先アドレスの検証
  4. メール送信APIを呼び出し
  5. 実行ログを出力

### 4.2 Cowork側のスケジュールタスク
- **タスク名**: `GetWeeklyScheduleAndFormatEmail`
- **実行トリガー**: GitHub Actionsからの手動呼び出し
- **処理内容**:
  1. 引数から実行日を取得
  2. Google Calendarから1週間分のデータを取得
  3. メール本文をMarkdown形式で生成
  4. JSONで結果を返却
- **出力フォーマット**:
  ```json
  {
    "success": true,
    "email_subject": "高見徹の１週間の予定（2026/3/21〜2026/3/27）",
    "email_body": "...",
    "recipient": "takami.sp@gmail.com",
    "date_generated": "2026-03-21T08:00:00+09:00"
  }
  ```

## 5. エラーハンドリング

| エラー箇所 | エラー内容 | 対応 |
|-----------|---------|------|
| Cowork API呼び出し失敗 | ネットワーク接続失敗など | リトライ（最大3回）、GitHub Actionsで警告通知 |
| カレンダーデータ取得失敗 | 認証エラー、APIエラー | GitHub Actionsで詳細ログ出力 |
| メール本文生成失敗 | データフォーマットエラーなど | GitHub Actionsで詳細ログ出力 |
| 送信先検証失敗 | アドレスが takami.sp@gmail.com でない | メール送信中止、警告ログ出力 |
| メール送信失敗 | Gmail API エラーなど | GitHub Actionsで詳細ログ出力 |

## 9. 付録

### 9.1 Google Calendar API の接続
- Cowork側で既に `gcal_list_events` ツールが利用可能
- 新たなAPI設定は不要

### 9.2 Gmail API の接続
- Cowork側で既に `gmail_create_draft` 等のツールが利用可能
- メール送信はCowork側で実施予定

### 9.3 GitHub Actions の設定
- リポジトリURL: （ユーザーのGitHubリポジトリURL）
- ワークフローファイル: `.github/workflows/weekly-schedule-email.yml`
- Secrets設定: Cowork API呼び出し用の認証情報（必要に応じて）
